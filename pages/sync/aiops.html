PostgreSQL pgvector Graph Relation

화면의 작은 변경이 어떤 API, DB 테이블, Mapper, 단위 테스트까지 파급되는지 유기적으로 추적할 수 있는 맵을 구현합니다.

code_file

db_schema_index, Table/Column, Entity 매핑 이력

semantic_idx

SyncOps 7대 핵심 운영 원칙

AI 자율 판단 범위를 인지하고 안정적 운영 품질을 담보하는 규칙

01

사전 분석 DB 기반 판단

매번 실시간 소스 전수 조사를 지양하고 구축·보관된 고속 메타데이터를 기초로 추론합니다.

02

담당자 승인 필수

AI의 완전 자율 배포는 배제하고, 사람이 검토·승인한 후에만 가동합니다.

03

승인 범위 내 수정 (Patch)

임의 소스 생성을 제한하고, 승인된 파일 군 내 지엽적 Patch만 적용합니다.

04

PR 기반 코드 반영

Main 브랜치 직접 덮어쓰기 없이 Pull Request로 생성 후 사람이 감시합니다.

05

자동 Merge 절대 금지

결함이 해소되어도 즉시 반영하지 않으며, 수동 리뷰와 관리자 승인이 선행됩니다.

06

품질 통제 게이트 의무화

SonarQube Bug·취약점·커버리지 기준 미달 시 완료 처리를 차단합니다.

07

Merge 후 DB 증분 갱신

최종 병합 시 Doc Agent가 갱신 내역을 분석해 다음 사이클 정밀도를 유지합니다.
