업무별 워크플로우를 만들고 실행 성공까지 확인하는 화면

각 데모는 실제 v2v 화면 캡처를 기반으로 구성했습니다. 빈 캔버스에서 블럭을 하나씩 놓고, 엣지를 연결하고, 실행 버튼을 눌러 성공 로그를 확인하는 흐름을 한 화면에서 크게 볼 수 있습니다.

ETL / 대량 데이터 BaaS API 웹훅 + 로컬 LLM CI/CD EAI / 시스템 연계 운영 모니터링
Data Work

ETL / 대량 데이터 처리

소스 데이터를 읽고 Bulk Scope 안에서 변환과 적재를 반복한 뒤 완료 결과를 기록하는 업무 데모입니다.

1업무 템플릿에서 ETL 대량 처리 선택
SELECT
2Source DB, Target DB, 배치 크기 입력
CONFIG
3Bulk Scope 안에 반복 처리 흐름 구성
SCOPE
4변환과 적재 블럭을 엣지로 연결
BUILD
5검증 결과에 따라 완료/오류 경로 분기
CHECK
6실행 버튼으로 처리 로그와 결과 확인
RUN
데모 화면클릭하면 크게 볼 수 있습니다.
Source 조회 → Bulk Scope 판단 → 소량/대량 분기 → 배치 검증 → 커밋/오류 리포트 → 실행 성공
API Gateway

BaaS API 업무 등록

내부 시스템이 호출할 API를 워크플로우로 만들고, 테스트 파라미터와 헤더를 입력해 실행 검증까지 진행하는 업무 데모입니다.

1API IN으로 라우트와 메서드 정의
INPUT
2M2M 인증 정책을 앞단에 연결
AUTH
3필수 파라미터 조건으로 TRUE/FALSE 분기
VALIDATE
4정상 요청은 DB 조회와 응답 DTO 매핑
MAP
5조회 결과에 따라 200/404 응답 분기
BRANCH
6테스트 팝업에서 Header / Body 입력 후 실행
RUN
데모 화면BaaS API
API IN → 인증 → 입력 검증 → 성공/실패 응답 분기 → 조회 결과 분기 → 실행 성공
AI Agent

웹훅 + 로컬 LLM 응답 서비스

웹훅 메시지를 받아 AIA가 직접 답변하거나 Tool Runner를 호출하고, 최종 결과를 다시 웹훅으로 발송하는 업무 데모입니다.

1Webhook IN으로 프롬프트 수신
INPUT
2메시지 존재 여부로 빈 요청 분기
CHECK
3AIA가 직접 답변 또는 도구 사용 판단
ROUTE
4도구 사용 시 대기 안내 메시지 발송
NOTICE
5AIA Scope 안에서 Tool Runner 반복 실행
TOOLS
6최종 답변을 Webhook OUT으로 반환
CALLBACK
데모 화면Webhook + Local LLM
Webhook IN → 메시지 검증 → 직접 응답/도구 사용 분기 → Tool Runner → 최종 콜백
DevOps

CI/CD 자동화

빌드, 소스 품질 체크, 조건 분기, 배포 명령, 결과 알림을 하나의 워크플로우로 연결하는 업무 데모입니다.

1소스 체크아웃과 빌드 명령 실행
BUILD
2단위 테스트와 소스 품질 체크 연결
QUALITY
3Quality Gate 조건으로 통과/실패 분기
GATE
4실패 시 배포 중단 리포트 발송
FAIL
5통과 시 배포와 Smoke API 테스트 실행
DEPLOY
6최종 결과를 운영 채널로 알림
REPORT
데모 화면CI/CD DevOps
Checkout → Build → Test → Quality Gate → 배포/실패 리포트 분기 → 결과 알림
Integration

EAI / 시스템 연계

여러 API와 DB를 연결하고 앞 단계 결과값을 다음 API 파라미터로 주입하는 시스템 연계 업무 데모입니다.

1ERP API에서 주문 데이터를 수집
FETCH
2파트너 DTO 형식으로 정규화
MAP
3스키마 검증 조건으로 정상/오류 분기
VALIDATE
4정상 데이터는 DB에 반영
UPSERT
5외부 전송 필요 여부로 전송/생략 분기
BRANCH
6전송 결과와 감사 이력을 기록
AUDIT
데모 화면EAI Integration
수집 → 정규화 → 스키마 검증 → DB 반영 → 외부 전송/생략 분기 → 감사 기록
메시지 이벤트

Kafka/MQ 이벤트 릴레이

Kafka, Redpanda, 메시지 큐에서 들어온 이벤트를 수신하고, 검증하고, 변환하고, 저장한 뒤 다시 다른 토픽으로 발행하는 과정을 하나의 v2v DAG 워크플로우로 구성합니다.

1Kafka/MQ 토픽 메시지를 수신합니다
수신
2이벤트 본문을 검증하고 조건으로 분기합니다
검증
3다음 시스템이 사용할 표준 메시지로 변환합니다
변환
4감사 로그 또는 업무 상태를 DB에 저장합니다
저장
5처리된 이벤트를 다른 토픽으로 발행합니다
토픽 발행
6실행 이력과 재처리 경로를 확인합니다
관측
데모 화면라이트 테마로 구성한 Kafka/MQ DAG 데모
Topic IN → 검증 → 변환 → DB 감사 저장 → Topic OUT → 실행 이력 확인
대량 트래픽 제어

Redis 기반 티켓 선착순 예매

대량 트래픽이 몰리는 티켓 구매 요청을 Redis Lua Script로 원자적으로 판단하고, v2v DAG 안에서 재고 차감, 대기 순번 부여, 중복 요청 방지, 알림과 실행 이력을 함께 관리합니다.

1BaaS/API로 티켓 구매 요청을 받습니다
요청
2Redis Lua Script로 재고와 대기열을 판단합니다
레디스
3확정, 대기, 중복, 매진 상태를 분기합니다
정책
4대기 고객에게 순번 조회 응답을 제공합니다
대기
5예약 결과와 실행 감사 이력을 저장합니다
저장
6트래픽 처리 결과를 운영 채널로 알립니다
보고
데모 화면라이트 테마로 구성한 Redis 티켓 예매 DAG 데모
API 요청 → Redis Lua 정책 → 확정/대기 분기 → 예약 저장 → 운영 알림
Operations

운영 모니터링과 복구

워크플로우 이력, 실패 로그, 재시동 이후 밀린 스케줄을 확인하고 운영자가 필요한 복구 실행을 선택하는 업무 데모입니다.

1SSH로 서버 CPU/Memory 정보를 수집
WATCH
2수집 값을 파싱해 운영 지표로 변환
PARSE
3임계치 조건으로 정상/장애 경로 분기
CHECK
4장애 시 진단 명령과 복구 가능 여부 판단
DIAGNOSE
5자동 복구 또는 수동 에스컬레이션 분기
RECOVER
6검증 결과와 복구 리포트를 발송
REPORT
데모 화면Operations
서버 지표 수집 → 임계치 판단 → 정상/복구 분기 → 자동 복구/수동 에스컬레이션 → 리포트
